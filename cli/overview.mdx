The `ahvn` CLI provides a familiar interface inspired by `conda` and `git` for managing LLM configurations, project repos, and AI agent workflows.

## Usage

```bash
ahvn [OPTIONS] COMMAND [ARGS]...
```

| Flag | Description |
|---|---|
| `-v`, `--version` | Show the AgentHeaven version and exit. |
| `-h`, `--help` | Show help and exit. |

<Tip>
All commands and subcommands support prefix-based aliases via `AliasedGroup`. You can type a unique prefix of any command name instead of the full name — for example, `ahvn co` instead of `ahvn config`.
</Tip>

## Command reference

### `ahvn setup`

Initialize AgentHeaven globally, similar to `conda init`. Creates the global config if it does not exist.

```bash
ahvn setup [--reset/-r]
```

| Flag | Description |
|---|---|
| `-r`, `--reset` | Reset the global configuration to default values. |

### `ahvn clean`

Remove all files in the configured `cache_path` and `tmp_path` directories.

```bash
ahvn clean [--dry-run/-n]
```

| Flag | Description |
|---|---|
| `-n`, `--dry-run` | Print what would be deleted without actually deleting anything. |

### `ahvn pj`

Join one or more path strings using `hpj()`, with proper handling of `~`, `&`, `>`, and forward slashes. Prints the resulting absolute path.

```bash
ahvn pj STRING [STRING ...]
```

```bash
ahvn pj "~/projects" "myrepo"
# /home/user/projects/myrepo
```

## Getting help

Append `--help` or `-h` to any command or subcommand to view its options:

```bash
ahvn --help
ahvn repo --help
ahvn config set --help
```
