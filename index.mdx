## Why AgentHeaven?

AgentHeaven brings a conda-inspired project management philosophy to AI agent development. Instead of wiring together LLM clients, vector stores, and caching layers manually, AgentHeaven provides a unified, batteries-included toolkit.

<CardGroup cols={2}>
  <Card title="LLM Abstraction" icon="brain">
    One interface for OpenAI, OpenRouter, DeepSeek, Ollama, and any LiteLLM-compatible provider. Switch models without changing code.
  </Card>
  <Card title="Knowledge Base" icon="database">
    KLStore + KLEngine + KLBase compose into a powerful knowledge management layer with vector, facet, MongoDB, and DAAC search engines.
  </Card>
  <Card title="Agentic Programming" icon="wand-magic-sparkles">
    `@autocode`, `@autofunc`, and `@autotask` decorators let LLMs implement, wrap, and orchestrate Python functions automatically.
  </Card>
  <Card title="Smart Caching" icon="bolt">
    Multiple cache backends (in-memory, disk, JSON, MongoDB) with a `memoize()` decorator for transparent LLM call caching.
  </Card>
  <Card title="Tool System" icon="wrench">
    `ToolSpec` converts Python functions into structured tool schemas ready for LLM function-calling.
  </Card>
  <Card title="Agent Framework" icon="robot">
    `BaseAgentSpec` provides a streaming-first, multi-step agent loop with tool use, user proxy, and configurable termination.
  </Card>

## Core components

```
ahvn/
├── llm/          LLM abstraction (oracle, stream, embed, tooluse)
├── cache/        Cache backends (InMemCache, DiskCache, JsonCache, MongoCache)
├── tool/         ToolSpec & ToolRegistry for function-calling
├── ukf/          Universal Knowledge Fragments (schema types)
├── klstore/      KL storage backends
├── klengine/     KL search engines (DAAC, Facet, Vector, Mongo, Scan)
├── klbase/       KLBase: unified storage + engine manager
├── agent/        Agent specs (BaseAgentSpec, ConvToolAgentSpec)
├── cli/          ahvn CLI (setup, config, repo, chat, session, embed)
└── utils/exts/   Agentic decorators (autocode, autofunc, autotask)
```

## Quick example

```python quickstart.py
from ahvn.llm import LLM
from ahvn.cache import InMemCache
from ahvn.tool import ToolSpec

# Single-turn LLM call
llm = LLM(preset="chat")
response = llm.oracle("Explain what a Fibonacci sequence is in one sentence.")
print(response)

# Function tool usage
def multiply(a: int, b: int) -> int:
    """Multiply two integers."""
    return a * b

tool = ToolSpec.from_function(multiply)

# Force the model to call the tool and get back the result
tool_messages = llm.tooluse("What is 42 * 37?", tools=[tool])
print(tool_messages)
# [{"role": "tool", "name": "multiply", "content": "1554", ...}]

# Memoized function with cache
cache = InMemCache()

@cache.memoize()
def fibonacci(n: int) -> int:
    """Compute the nth Fibonacci number."""
    return n if n <= 1 else fibonacci(n - 2) + fibonacci(n - 1)

print(fibonacci(32))  # cached after first call
```
